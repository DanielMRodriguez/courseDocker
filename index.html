Docker Capacitación

Docker es una plataforma de contenedores que permite a los desarrolladores crear, desplegar y gestionar aplicaciones de forma eficiente y escalable.

Arquitectura de Docker

Principales conceptos

  • Contenedor

    Un contenedor es una instancia de una imagen que se ejecuta en un host. Es el entorno de ejecución de una aplicación.

  • Imagen

    Una imagen es una plantilla para crear contenedores. Es el estado de un contenedor en un momento dado.

  • Volumen

    Un volumen es un espacio de almacenamiento persistente que se puede montar en un contenedor. Es el estado de un contenedor en un momento dado.

  • Red

    Una red es un conjunto de contenedores que se pueden comunicar entre sí. Es el estado de un contenedor en un momento dado.

  • Dockerfile

    Un Dockerfile es un archivo que contiene las instrucciones para crear una imagen. Es el estado de un contenedor en un momento dado.

Contenedor de Docker

Comandos básicos

docker pull

Uso:

docker pull [OPTIONS] NAME[:TAG|@DIGEST]

Descarga una imagen desde un registro.

Alias: docker image pull, docker pull

Opciones comunes:
  • -a, --all-tags
      Descargar todas las etiquetas de la imagen en el repositorio
  • --disable-content-trust   Omitir la verificación de la imagen (por defecto true)
  • --platform string   Establece la plataforma si el servidor es multiarquitectura
  • -q, --quiet   Suprime la salida detallada

Ejemplo:

docker pull ubuntu:latest
Esto descargara la ultima imagen de la version ubuntu

Si ya estas usando una imagen de ubuntu, puedes usar docker pull para asegurarte de que tengas la ultima version.

docker images

Lista las imágenes disponibles localmente en tu máquina Docker.

Alias: docker image ls, docker image list, docker images

Uso:
docker images [OPTIONS] [REPOSITORY[:TAG]]

Opciones comunes:
  • -a, --all
      Muestra todas las imágenes (incluidas las intermedias)
  • --digests
      Muestra los identificadores digest de las imágenes
  • -f, --filter [filtro]
      Filtra la salida según condición (por ejemplo, dangling=true para ver imágenes huérfanas)
  • --format [string]
      Personaliza la salida usando plantillas Go (ver la documentación oficial)
  • --no-trunc
      No trunca la salida (muestra los hashes completos)
  • -q, --quiet
      Solo muestra los IDs de imagen
  • --tree
      Muestra las imágenes multiplataforma como árbol (Experimental)

Ejemplo:

docker images
Muestra la lista de imágenes locales y sus detalles.

Filtrar imágenes:

docker images -f "dangling=true"
Muestra solo las imágenes huérfanas ("dangling"), útiles para limpiar el sistema.

Obtener solo IDs:

docker images -q
Esta opción es útil para pasar IDs de imágenes a otros comandos, como docker rmi.

Puedes consultar la documentación oficial de Docker para ver más ejemplos y todos los filtros y formatos disponibles para docker images.

docker ps

El comando docker ps se usa para listar los contenedores activos en tu sistema Docker. Por defecto, solo muestra los contenedores que están en ejecución, pero puedes ver todos (incluidos los detenidos) y personalizar la salida mediante varias opciones.

  • Básico:
    docker ps
    Lista solo los contenedores en ejecución.
  • Mostrar todos los contenedores (incluso los detenidos):
    docker ps -a
    Muestra todos los contenedores, incluso los que están detenidos.
  • Ver únicamente IDs de contenedores:
    docker ps -q
    Útil para scripts donde solo se necesitan los identificadores.
  • Ver solo el último contenedor creado:
    docker ps -l
    Muestra únicamente el último contenedor que fue creado.
  • Ver los n últimos contenedores creados:
    docker ps -n 5
    Muestra los últimos 5 contenedores, sin importar su estado.
  • Filtrar por criterio (por ejemplo, estado):
    docker ps -f "status=exited"
    Solo muestra los contenedores detenidos (exited).
  • Mostrar el tamaño que ocupan los archivos del contenedor:
    docker ps -s
    Incluye el tamaño en disco usado por cada contenedor.
  • Personalizar las columnas de la tabla con formato:
    docker ps --format "table {{.ID}}\t{{.Image}}\t{{.Status}}"
    Personaliza las columnas que se muestran en la salida, usando plantillas de Go.
  • Ver la salida en formato JSON:
    docker ps --format json
    Muestra la información en formato JSON (útil para procesamiento programático o en herramientas externas).

Puedes consultar la documentación oficial de Docker para ver más ejemplos, todas las opciones y filtros disponibles para

docker ps.
